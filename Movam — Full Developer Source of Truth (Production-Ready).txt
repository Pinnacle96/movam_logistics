4. Payment & Wallet Architecture
* Prepaid only (no cash-on-delivery)
* Customer pays Movam → funds split automatically
* Commission deducted at source
* Wallet system for riders and merchants:
   * Rider Wallet: delivery fee payout
   * Merchant Wallet: order value minus commission
   * Movam Wallet: commission + delivery margin + subscription revenue
* Refunds & disputes handled via Laravel backend
* Payment APIs: Paystack / Flutterwave (PCI compliant)
Example Flow
Customer pays ₦11,000 (₦10,000 order + ₦1,000 delivery)
* Merchant commission 12% → ₦1,200 to Movam
* Rider delivery share 70% → ₦700
* Movam margin: commission + 30% delivery fee → ₦1,500 total


5. Real-time & High Concurrency
* Laravel Octane + Swoole → thousands of concurrent requests
* RabbitMQ → asynchronous queues for:
   * Order assignment
   * Wallet settlements
   * Notifications
   * Subscription renewals
* Redis optional for caching & real-time location updates
* WebSockets via Laravel WebSockets (with Octane)


6. Infrastructure & Deployment (Cost-Optimized)
* Backend: Dockerized Laravel 11+ Octane API
* Queue Broker: RabbitMQ self-hosted (free)
* Cache: Optional Redis for small instance
* Database: MySQL 8 / PostgreSQL 15
* Storage: AWS S3 / DigitalOcean Spaces
* Mobile: Flutter 3.x builds
* Admin Console: Vue.js 3 + TailwindCSS
* CI/CD: GitHub Actions, automated Docker builds, automated deployment to staging/production
* SSL & HTTPS: Nginx with Let’s Encrypt
Cost Reduction Measures
* Use Mapbox free tier for maps & routing
* Self-host RabbitMQ instead of managed queues
* Redis only for caching, not queue broker
* DigitalOcean Spaces for cost-effective storage
* Prepaid-only payment avoids cash handling overhead

4. Run migrations & seeders

6. Install Flutter SDK + Android Studio + Xcode (iOS)
7. Configure Firebase for FCM notifications
8. Configure payment APIs (Paystack / Flutterwave)
9. Setup RabbitMQ & optional Redis
10. Configure cron jobs for wallet settlement, subscription renewals, analytics


9. Mandatory Functional Flow
1. Customer places order → prepaid payment
2. Laravel validates & saves order
3. Order queued in RabbitMQ → triggers:
   * Rider assignment
   * Merchant notification
   * Customer notification
   * Wallet settlement
4. Rider picks up & delivers → confirms via app
5. Wallets updated automatically
6. Notifications sent at each status change
7. Analytics updated in real-time


10. iOS & Android App Considerations
* iOS: APNs + FCM, background location tracking, App Store guidelines, PCI compliant payments
* Android: FCM, GPS tracking, background services for rider, Play Store deployment
* Prepaid only → simplifies compliance
* Offline caching for poor connectivity areas


11. Revenue Enforcement
* Mandatory monetization on all streams:
   * Merchant commission
   * Delivery fee margin
   * Priority listings / promotions subscription
   * Logistics-as-a-Service (LaaS) for enterprises
   * Data insights & analytics for retailers
* All transactions handled in platform → automatic split of funds
* Prepaid ensures revenue is captured before delivery
